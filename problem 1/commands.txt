Внутренняя память виртуальной машины устроена следующим образом:
1.Системные регистры:
    IP указатель на текущую инструкцию
    COMPARISON_FLAG флаг результатов операции сравнения (= 0, < 1, > 2)
    SP указатель на вершину стэка
    CUR_COMMAND обрабатываемая команда
    SRC_ARG параметр источника
    DST_ARG параметр приемника
    WORK_STATUS статус работы программы
2.Код программы: блок данных, инструкции (ячейки по 4 байта).
3.Стэк (размер 65536 ячеек)

Дизассемблер будет работать корректно, т. к. блок статических данных всегда предваряет блок инструкций,
которые начинаются с позиции, записанной в регистр IP.

Доступные команды:
-1      NONE                пустая команда
255     END                 меняет WORK_STATUS на false

2       MOV [src] [dst]     копирует значение из src в dst
3       ADD [src] [dst]     добавляет значение из src к dst

4       CMP [src] [dst]     сравнивает src и dst, пишет результат в COMPARISON_FLAG

5       OUT [src]           выводит на экран значение из src
6       IN [dst]            считывает с клавиатуры значение в dst

7       JNQ [dst]           переводит IP в dst, если результат сравнения !=
8       JL [dst]            переводит IP в dst, если результат сравнения <
9       JQ [dst]            переводит IP в dst, если результат сравнения =
10      JG [dst]            переводит IP в dst, если результат сравнения >
11      JMP [dst]           переводит IP в dst

12      CALL [dst]          пишет в стэк адрес следующей ячейки, переводит IP в dst
13      RET                 возвращается по адресу, лежащему на вершине стэка
14      PUSH [src]          кладет в стэк значение из src
15      POP [dst]           достает из стэка значение в dst
